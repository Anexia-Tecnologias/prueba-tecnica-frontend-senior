# Prueba técnica para desarrollador frontend en Alquilerseguro

## Contexto

Alquilerseguro quiere renovar su página web y el arquitecto de frontend (_Pepe_) ha creado un **nuevo proyecto con React, TypeScript y Astro para el nuevo desarrollo de Alquilerseguro**.

_Pepe_ ha dejado una estructura y algunos componentes a modo de ejemplo para que el resto de desarrolladores puedan continuar siguiendo la línea, y se ha comenzado implementando la tarea:

- [Tarea 1: Área privada: login y listado de viviendas alquiladas](./.user-stories/01-area-privada.md)

Durante el desarrollo, ha habido algunos cambios en el equipo:

- _Pancho_, el desarrollador frontend que comenzó la tarea, ya no está en el equipo.
- _Francisca_ se ha unido al equipo como nueva desarrolladora frontend, y es la encargada de continuar con la tarea que _Pancho_ dejó a medias.
- _Pepe_ ha sido ascendido a CEO de la empresa y ya no está activamente involucrado en el desarrollo.

El equipo ha decidido contratar a un nuevo **Desarrollador Frontend Senior** para velar por la calidad del código, la experiencia de usuario y la seguridad de la aplicación.

Tu objetivo es revisar la implementación que _Francisca_ ha realizado en la rama `feature/01-area-privada`, refactorizar el código para mejorar su calidad y explicar a _Francisca_ los cambios que has realizado y por qué, para que pueda aprender de ellos y aplicarlos en el futuro.

## Instrucciones para completar la prueba técnica

1. Deberás importar el repositorio de la prueba en tu propio repositorio de GitHub, manteniéndolo como repositorio privado [Importar repositorio](https://github.com/new/import).

2. Abre una nueva rama a partir de `feature/01-area-privada` con el nombre `feature/01-area-privada--refactor`,
    - Realiza los cambios que consideres necesarios para mejorar la calidad del código, la experiencia de usuario y la seguridad de la aplicación.
    - Explica y razona todas las decisiones que has tomado en los cambios realizados, para que _Francisca_ pueda aprender de ellos. Puedes usar el medio que consideres más adecuado para ello.
    - Usa las herramientas de GitHub que consideres necesarias para cumplir con el objetivo de la prueba técnica.

3. Antes de que cumpla el plazo informado en el correo en el que has recibido esta prueba,
    - Invita a [asaldana-anexia](https://github.com/asaldana-anexia) como colaborador de solo lectura a tu repositorio de GitHub, para que podamos revisar tu solución.
    - Envía un email:
        - Con el asunto: "Prueba técnica frontend - [Tu nombre]"
        - Para: `asaldana@anexia.es`
        - CC: `spinos@anexia.es`
        - Cuerpo del mensaje: **enlace a tu repositorio de GitHub**

## Se valorará:

- Capacidad de análisis y razonamiento para identificar áreas de mejora en el código.
- Identificar líneas de arquitectura previas para mantener la coherencia con el proyecto.
- Capacidad para equilibrar la propuesta de cambios con la arquitectura existente.
- Habilidad para refactorizar el código de manera efectiva preservando y/o mejorando la funcionalidad existente.
- Capacidad para explicar modificaciones con claridad a perfiles con menos experiencia.
- Conocimiento de las herramientas de GitHub para la gestión del flujo de trabajo y la colaboración en equipo.
